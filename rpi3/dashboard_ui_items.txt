# RPI3 Dashboard UI/UX 표시 항목 정리

목표:
- 한눈에 주차장 상태를 파악할 수 있게 한다.
- 실제 제어 상태와 MQTT 수신 상태를 구분해서 보여준다.
- 현재 MQTT 계약으로 확인 가능한 정보만 우선 표시한다.

---

## 1. 상단 요약 카드

### Parking Lot
- 표시값: AVAILABLE / FULL
- 의미:
  - AVAILABLE: 빈 슬롯 있음
  - FULL: 모든 슬롯 점유
- source:
  - topic: parking/rpi1/lot
  - payload: AVAILABLE 또는 FULL

### Available Slots
- 표시값: 0 / 1 / 2
- 의미:
  - Slot 상태 기준으로 비어 있는 슬롯 수
- source:
  - topic: parking/rpi1/slot
  - payload 예시: {"slot1":"EMPTY","slot2":"OCCUPIED"}

### Gate
- 표시값: OPEN / CLOSED
- 의미:
  - OPEN: 게이트 열림
  - CLOSED: 게이트 닫힘
- source:
  - topic: parking/rpi2/gate
  - payload: OPEN 또는 CLOSED

### Dashboard MQTT
- 표시값: CONNECTED / SUBSCRIBED / RECONNECTING / DISCONNECTED / ERROR
- 의미:
  - 브라우저 Dashboard가 Mosquitto WebSocket에 연결된 상태
- source:
  - browser mqtt.js client 상태
- 주의:
  - 이 값은 RPI1/RPI2의 실제 online 상태가 아니다.

---

## 2. 주차 슬롯 표시

### Slot 1
- 상태:
  - EMPTY: 차량 없음
  - OCCUPIED: 주차 완료, 점유
- 거리:
  - cm 단위 초음파 거리값
- source:
  - 상태 topic: parking/rpi1/slot
  - 거리 topic: parking/rpi1/distance

### Slot 2
- 상태:
  - EMPTY: 차량 없음
  - OCCUPIED: 주차 완료, 점유
- 거리:
  - cm 단위 초음파 거리값
- source:
  - 상태 topic: parking/rpi1/slot
  - 거리 topic: parking/rpi1/distance

### 시각 표현
- EMPTY:
  - 초록 계열
  - 차량 아이콘 없음
- OCCUPIED:
  - 빨강 계열
  - 차량 아이콘 표시
- UNKNOWN:
  - 회색 계열
  - 초기 상태 또는 데이터 없음

---

## 3. 게이트 표시

### Gate State
- 표시값: OPEN / CLOSED
- source:
  - topic: parking/rpi2/gate

### Last Gate Event
- 표시값:
  - ENTRY_OPEN
  - EXIT_OPEN
- 의미:
  - ENTRY_OPEN: 진입으로 인한 게이트 open
  - EXIT_OPEN: 출차로 인한 게이트 open
- source:
  - topic: parking/rpi2/event
  - payload 예시: {"event":"ENTRY_OPEN","reason":"vehicle_detected"}

### 차량 접근/출차 요청 여부
- 현재 별도 상태 topic은 없음.
- 현재는 event로 간접 표시한다.
  - ENTRY_OPEN 수신 시: 진입 차량 접근 감지로 판단
  - EXIT_OPEN 수신 시: 출차 스위치 입력으로 판단
- 지속 상태로 보여주려면 RPI2에서 별도 topic이 필요하다.
  - 예: parking/rpi2/entry_sensor
  - 예: parking/rpi2/exit_switch

## 4. MQTT / Node 상태

### 현재 표시 가능한 것
- Dashboard MQTT 연결 상태
- 마지막 수신 topic
- 마지막 수신 payload

### 최종 UI에서 권장
- 일반 화면:
  - Dashboard MQTT: SUBSCRIBED 정도만 표시
- 개발/디버그 화면:
  - Last MQTT Topic
  - Last MQTT Payload

### RPI1/RPI2/RPI3 online 상태
- 현재 MQTT 계약만으로는 정확히 표시할 수 없다.
- 이유:
  - broker가 Dashboard에 각 node의 생존 여부를 자동으로 알려주지 않음
  - 마지막 메시지가 오래됐다고 해서 반드시 offline이라고 단정할 수 없음

### online 상태를 표시하려면 추가 topic 필요

권장 topic:
- parking/rpi1/status
- parking/rpi2/status
- parking/rpi3/status

payload 예시:
```json
{
  "node": "rpi1",
  "status": "online",
  "ip": "192.168.0.11"
}
```

판단 기준:
- 마지막 status 수신 후 5초 이내: ONLINE
- 마지막 status 수신 후 5초 초과: STALE
- 마지막 status 수신 후 15초 초과: OFFLINE

현재 단계에서는 heartbeat topic을 구현하지 않았으므로 최종 UI에 RPI1/RPI2 online 상태를 단정 표시하지 않는다.

## 5. 이벤트 로그

표시 항목:
- timestamp
- event
- reason
- topic

표시 대상:
- ENTRY_OPEN
- EXIT_OPEN

표시 예:
- [2026-06-03 02:33:01] EXIT_OPEN (switch_pressed)
- [2026-06-03 02:32:58] ENTRY_OPEN (vehicle_detected)

정렬:
- 최신 이벤트가 위로 오게 표시

## 6. 우선 UI 구성안

### 상단
- Available Slots
- Parking Lot
- Gate
- Dashboard MQTT

### 중앙
- 주차장 top-down view
- Slot 1 / Slot 2
- Gate

### 우측 패널
- Slot Details
  - Slot 1 상태, 거리
  - Slot 2 상태, 거리
- Gate Details
  - Gate 상태
  - Last Gate Event
- Event Log

### 개발용 접이식 영역
- Last MQTT Topic
- Last MQTT Payload

---

## 7. 현재 UI에서 제거 또는 축소할 항목

- Last MQTT Topic:
  - 개발 중에는 유지
  - 최종 UI에서는 접이식 debug 영역으로 이동
- Last MQTT Payload:
  - 개발 중에는 유지
  - 최종 UI에서는 접이식 debug 영역으로 이동
- 큰 격자 배경:
  - 현재는 공간이 넓고 정보 밀도가 낮음
  - 슬롯/게이트가 더 눈에 띄도록 비율 조정 필요

---

## 8. 다음 UI 수정 방향

1. 상단 카드 재구성
2. 중앙 주차장 영역에서 Slot/Gate 크기와 위치 조정
3. 우측 패널을 Slot / Gate / Event 섹션으로 분리
4. 디버그 MQTT 정보는 접이식 영역으로 이동
5. 색상 의미 통일
   - EMPTY / AVAILABLE / OPEN: 초록
   - OCCUPIED / FULL / CLOSED: 빨강 또는 강조색
   - UNKNOWN / DISCONNECTED: 회색
