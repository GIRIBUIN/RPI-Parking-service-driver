# 개발 스케줄

---

###  Rpi #1 담당자: 주차 공간 센싱 및 경고 노드

**[1주 차] 하드웨어 결선 및 개별 센서 제어 테스트**
* **Task 1.1**: Raspberry Pi OS 설치 및 SSH/VS Code 원격 개발 환경 구축.
* **Task 2.1**: 초음파 센서(HC-SR04) 3개(정면, 후면, 측면) 브레드보드 결선 및 개별 거리 측정 Python 스크립트 작성.
* **Task 2.2**: 센서 간섭을 방지하기 위해 3개의 센서를 **순차적으로 측정**하는 딜레이(Delay) 로직 필수 구현.
* **Task 3.1**: LED 및 Buzzer 결선 후 GPIO 제어 테스트 (점등, 소등, 깜빡임, Beep 음 등).

**[2주 차] 상태 판별 알고리즘 구현 및 통신 준비**
* **Task 1.1**: 측정된 거리를 바탕으로 주차 공간 상태(`EMPTY`, `APPROACHING`, `PARKING`, `PARKED`) 판별 상태 머신(State Machine) 로직 구현.
* **Task 1.2**: 거리별 위험도(`SAFE`, `WARNING`, `DANGER`) 임계값(Threshold) 설정.
* **Task 2.1**: 위험도 상태에 따른 경고 액션 매핑. 
  * `WARNING`: LED 깜빡임, Buzzer 간헐적 beep.
  * `DANGER`: LED 점등 유지, Buzzer 연속 경고음.
* **Task 3.1**: Rpi #3 브로커로 주차 상태, 거리값, 위험도를 전송할 `paho-mqtt` 퍼블리셔(Publisher) 스크립트 작성.
* **Task 3.2**: `PARKED` 상태 판단 시 `parking/control/gate` 토픽으로 `CLOSE` 명령을 발행하는 로직 추가.

**[3주 차] 프로토타입 모형 적용 및 연동 테스트**
* **Task 1.1**: 폼보드로 제작된 50 x 30 cm (벽 높이 15cm) 크기의 물리적 모형에 센서 3개 부착 및 배선 정리.
* **Task 2.1**: 실제 15 x 8 cm 크기의 자동차 모형을 손으로 움직여가며, 센서가 차량의 진입부터 주차 완료까지의 시나리오를 정상적으로 인식하고 MQTT로 발행하는지 테스트.
* **Task 3.1**: Rpi #2(게이트)와의 연동을 위해 `PARKED` 및 `EMPTY` 상태 전환 시 메시지 전달 지연 시간(Latency) 체크.

**[4주 차] 데이터 필터링 및 안정화**
* **Task 1.1**: 초음파 센서 특유의 튀는 데이터(Noise)를 잡기 위해 이동평균필터(Moving Average Filter) 또는 중앙값(Median) 필터링 로직 추가.
* **Task 2.1**: 장시간 구동 테스트 및 예외 처리 (네트워크 연결 끊김 시 센서 및 경고 장치 초기화 로직 등).

###  Rpi #2 담당자: 바리게이트 및 출차 제어 노드

**[1주 차] 게이트 모터 및 스위치 입출력 제어**
* **Task 1.1**: Raspberry Pi OS 설치 및 원격 개발 환경 구축.
* **Task 2.1**: 서보 모터(Servo Motor) PWM 제어 스크립트 작성. 물리적인 게이트가 열리고(`OPEN`) 닫히는(`CLOSED`) 정확한 모터 각도 값(예: 0도, 90도) 도출 및 세팅.
* **Task 3.1**: 출차 요청용 Switch 버튼 결선. GPIO 풀업/풀다운 저항을 활용하여 버튼 클릭 시 하드웨어 인터럽트(Interrupt)가 발생하도록 코드 작성.
* **Task 3.2**: 게이트 Open/Close 상태를 표시할 외부 LED 점등 테스트.

**[2주 차] 시나리오 기반 제어 로직 및 MQTT 연동**
* **Task 1.1**: Rpi #3의 MQTT 브로커에 연결하여 게이트 제어 토픽(`parking/control/gate`)을 구독(Subscribe)하는 코드 작성.
* **Task 2.1**: 게이트 제어 시나리오 코드 적용.
  * 기본 상태: 모터 `OPEN` 유지.
  * `parking/control/gate` 토픽에서 `CLOSE` 명령 수신 시: 모터 `CLOSED` 제어.
* **Task 3.1**: 출차 Switch 버튼 클릭 시 즉시 모터를 `OPEN`으로 제어하고, 출차 요청 이벤트를 MQTT로 퍼블리시하는 기능 구현.
* **Task 3.2**: 게이트 상태(`OPEN`/`CLOSED`) 변경 시 `parking/rpi2/gate` 토픽으로 상태를 퍼블리시하는 기능 구현.

**[3주 차] 게이트 모형 조립 및 1차 통합 테스트**
* **Task 1.1**: 물리적 바리게이트 모형에 서보 모터와 스위치를 단단히 고정.
* **Task 2.1**: 스위치 바운싱(Bouncing, 한 번 눌렀을 때 여러 번 눌린 것으로 인식되는 현상) 방지를 위한 디바운싱(Debouncing) 소프트웨어 처리 추가.
* **Task 3.1**: Rpi #1 담당자와 함께 차량 진입 -> 주차 완료(게이트 닫힘) -> 출차 버튼(게이트 열림) -> 이탈 확인(EMPTY 초기화)에 이르는 전체 사이클 테스트.

**[4주 차] 안정성 확보 및 시연 준비**
* **Task 1.1**: 모터 과부하 방지 로직 (특정 각도 도달 시 PWM 신호 차단 등) 점검.
* **Task 2.1**: 통신 장애 시 게이트를 안전 상태(`OPEN`)로 유지하는 Failsafe 로직 구현.

###  Rpi #3 담당자: 중앙 서버 및 Digital Twin 대시보드

**[1주 차] 백엔드/프론트엔드 환경 구축 및 브로커 세팅**
* **Task 1.1**: Raspberry Pi OS 설치 및 Eclipse Mosquitto(MQTT Broker) 설치/구동.
* **Task 2.1**: Python FastAPI 프로젝트 초기화 및 SQLite 데이터베이스 스키마 설계 (센서별 측정 거리, 주차 상태, 위험도, 게이트 상태 기록용).
* **Task 3.1**: React.js + Tailwind CSS 프론트엔드 프로젝트 생성.
* **Task 3.2**: React 웹 앱 내에 `mqtt.js` 클라이언트를 연동하여 로컬 브로커의 테스트 메시지를 브라우저 콘솔에서 수신하는지 확인.

**[2주 차] 데이터 로깅 API 및 대시보드 UI 스캐폴딩**
* **Task 1.1**: FastAPI 백엔드에 MQTT 구독 데몬(Daemon)을 백그라운드로 띄워, 수신되는 모든 센서 및 상태 데이터를 DB에 타임스탬프와 함께 저장하는 로직 구현.
* **Task 2.1**: React 대시보드 레이아웃 구성. 좌측(70%)은 2D Canvas 주차장 영역, 우측(30%)은 실시간 데이터 모니터링 패널로 분할.
* **Task 3.1**: 우측 패널에 Rpi #1, #2에서 들어오는 주차 상태, 센서 거리, 게이트 개폐 상태를 실시간 텍스트로 렌더링.

**[3주 차] 2D Digital Twin 렌더링 구현 (핵심)**
* **Task 1.1**: HTML5 Canvas 또는 CSS를 활용하여 50 x 30 cm 주차장 비율에 맞는 2D Top-down 뷰 렌더링.
* **Task 2.1**: 수신되는 초음파 센서 거리 데이터를 역산하여, 화면 상의 15 x 8 cm 자동차 모형(UI 컴포넌트) 위치를 실시간으로 이동시키는 알고리즘 적용.
* **Task 3.1**: 대시보드 시각적 경고 효과 적용. 상태 토픽에 따라 대시보드 배경색 또는 테두리를 변경 (위험 시 붉은색 점멸 등).
* **Task 3.2**: 화면 좌측 하단에 바리게이트 아이콘을 배치하고, Rpi #2의 상태값에 따라 문이 열리고 닫히는 UI 애니메이션 연동.

**[4주 차] 렌더링 최적화 및 최종 데모 리허설**
* **Task 1.1**: React 렌더링 최적화 (상태값이 매우 빠르게 들어올 때 화면이 버벅이지 않도록 `requestAnimationFrame` 등 활용).
* **Task 2.1**: UI 디테일(모형과 싱크로율) 조정 및 버그 수정.
* **Task 3.1**: Rpi 1, 2 담당자와 함께 물리적 모형 위에서 자동차를 움직이며, 웹 화면의 Digital Twin이 오차율을 최소화하여 완벽히 동기화되는지 최종 점검.
